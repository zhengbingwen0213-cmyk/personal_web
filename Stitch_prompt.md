# Stitch Prompt

## Main Branch Web Style Summary

The current `main` branch is a restrained personal portfolio landing page built as a Vite + React single-page site. Its visual language is minimal, structured, and typography-led:

- Warm off-white paper background with a subtle dotted grid texture.
- Near-black text and borders, with very limited use of gray for secondary copy.
- A sticky top navigation bar with a compact `LB` monogram mark, centered navigation links, and a language toggle.
- A large editorial hero section with oversized type, strong negative space, and a right-side identity panel instead of a real portrait.
- Sharp rectangular geometry, thin borders, no rounded corners, and no decorative shadows.
- Content organized into clear portfolio sections: Home, About, Experience, Work, Skills, and Contact.
- Numeric section indexes, gridded stats, bordered timeline rows, simple project cards, and tag-like skill chips.
- Responsive behavior that stacks the hero, timeline, cards, and stats on mobile while preserving the calm monochrome layout.

## English Prompt To Paste Into Stitch

Use the existing main-branch site structure as the content foundation, but redesign it in a more refined **Minimalist Monochrome** editorial style. The prototype should feel like a high-end architecture monograph, museum catalog, or luxury editorial portfolio: austere, authoritative, timeless, intellectual, dramatic, and highly typographic.

Do **not** use any photos, portrait images, avatar placeholders, stock images, or generated character imagery. Replace all image areas with typography, monograms, bordered geometric frames, hairline grids, oversized initials, section numbers, rules, and abstract black-and-white layout devices.

### Site Structure

Design a one-page personal website with these sections:

1. Sticky header
   - Left: square `LB` monogram wordmark.
   - Center or right: navigation links for Home, About, Experience, Work, Skills, Contact.
   - Include a compact language toggle, but keep it visually understated.

2. Hero
   - Use oversized editorial typography as the primary visual element.
   - The name and intro should dominate the first viewport.
   - Include the role as a small uppercase label.
   - Include the summary as a refined body paragraph.
   - Add a large bordered identity panel or typographic poster block using `LB`, section lines, and geometric rules instead of a portrait.
   - Include one thick horizontal rule and one small bordered square as visual punctuation.

3. About
   - Use a two-column editorial layout: section index/title on the left, large readable statement on the right.
   - Add a boxed stats grid with three cells for Agent, AI, and PM.

4. Experience
   - Use a clean timeline or resume table with strong horizontal rules.
   - Each row should show period, organization, and description.

5. Work
   - Use two bordered project cards.
   - Cards should be flat, sharp, and monochrome.
   - Use hover-style inversion conceptually: black background with white text for active or highlighted states.

6. Skills
   - Use a black inverted section with white text.

7. Contact
   - End with a bold typographic call-to-action.

   - Keep the layout sparse, confident, and direct.

### Visual Style

Follow these design rules strictly:

- Palette: pure white `#FFFFFF`, pure black `#000000`, and limited neutral grays only for secondary text and hairline dividers.
- No accent colors. Black is the accent.
- Typography is the main visual asset.
- Use a refined serif typeface for display headings, such as Playfair Display or a similar high-contrast editorial serif.
- Use a readable serif for body copy, such as Source Serif 4 or Georgia.
- Use a monospace face for metadata, section numbers, dates, and labels.
- Use very large type in the hero, with at least one word or monogram feeling poster-sized.
- Use sharp 90-degree corners everywhere. Border radius must be 0.
- Use lines, borders, rules, underlines, and grids instead of shadows, gradients, or soft cards.
- Use no drop shadows.
- Use generous negative space and precise alignment.
- Add subtle paper-like texture, horizontal hairline pattern, or grid texture at very low opacity so the page does not feel flat.
- Use black inverted sections sparingly for emphasis, especially the Skills or final Contact area.
- Use instant or very fast interactions only: color inversion, underline reveal, border-thickness change. No bouncy animations, parallax, or soft easing.

### Layout And Responsiveness

- Desktop: use a max-width editorial container around 1120-1200px.
- Use a 12-column or strong two-column grid for hero and content sections.
- Separate major sections with thick black horizontal rules.
- Mobile: stack columns cleanly, reduce hero typography while preserving the dramatic monochrome feel, keep all touch targets at least 44px tall, and avoid overcrowding.
- Ensure no text overlaps, no labels overflow their containers, and every section remains readable on mobile.

### Accessibility

- Maintain strong contrast.
- Include visible focus states for links, buttons, and toggles.
- Use semantic section hierarchy.
- Keep navigation clear and scannable.
- Do not rely on color alone to communicate state.

### Desired Output

Produce a polished desktop and mobile prototype for the personal website. The result should look like a serious editorial portfolio for an AI product professional: minimal, monochrome, typographic, structured, and premium. It should not look like a generic startup landing page, a colorful tech template, or a portfolio with photo/avatar imagery.
